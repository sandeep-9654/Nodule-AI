Project Abstract

02

This project introduces an end-to-end web-based application designed for the automated detection and malignancy classification of pulmonary lung nodules from 3D CT scans. Driven by a highly optimized, CPU-efficient Lightweight 3D U-Net Architecture, it addresses the massive computational bottlenecks of traditional volumetric analysis while completely eliminating the need for expensive hospital GPUs.

By integrating a Next.js responsive frontend with a PyTorch backend and real-time WebGL rendering, the system empowers clinicians to instantly visualize critical spatial relationships in 3D, accelerating life-saving diagnostic workflows.

The Global Challenge

03
  • The Global Challenge: Lung cancer causes millions of deaths annually; early detection of pulmonary nodules is the absolute key to survival.
  • Current Bottleneck: Manual inspection of massive 3D CT scans slice-by-slice is slow, highly repetitive, and inherently prone to human fatigue.
  • Diagnostic Delays: Overwhelmed radiology departments face severe backlogs, meaning critical tumors can grow before a scan is ever reviewed.
  • High Inter-Observer Variability: Different doctors often interpret complex, irregular borders differently, leading to inconsistent biopsy decisions.
  • Our Solution: An automated, deeply-integrated AI diagnostic assistant for real-time, highly accurate nodule detection and classification directly inside a web browser.

The Obstacles in Medical AI

04

Model Bias

Extreme class imbalance in medical data (millions of healthy voxels vs. few tumor voxels) causes severe false-benign predictions.

Hardware Exhaustion

Full 3D isotropic volumes crash standard clinical workstations. Heavy convolutions demand non-existent local VRAM.

Data Overhead

Legacy datasets like LUNA-21 require massive, brittle local preprocessing pipelines just to extract regions of interest.

The "Black Box"

Traditional AI outputs a single number and lacks visually intuitive, explainable interfaces that a doctor can actually trust.

Why Current Methods Fall Short

05
  • Traditional CAD Systems: Rely completely on hand-crafted features (gradients, shapes). This leads to brittle generalization on new scanner data and high false-positive biopsy rates.
  • Heavyweight 3D CNNs (V-Net, Standard 3D U-Net): Produce great metrics but demand 24GB+ VRAM enterprise GPUs, making local hospital deployment financially and technically unscalable.
  • Manual Radiology Review: 2D planar viewing on traditional software misses the holistic 3D contextual relationships and margins of the tumor entirely.

System Architecture

A Paradigm Shift in Diagnostic AI

07
  • Strategic Dataset Migration: Transitioned the entire pipeline to the LUNA-22 (ISMI) 3D Patches dataset. Pre-extracted regions completely eliminate class imbalance.
  • Lightweight Architecture: Built a custom Lightweight U-Net 3D specifically tailored for CPU inference.
  • Drastic Compression: Total parameter count systematically reduced to ~215K (under 1 MB file size).
  • Full-Stack Deployment: A Next.js responsive frontend married to a high-speed Flask PyTorch backend.

The Lightweight U-Net 3D Model

13
  • The Foundation: The U-Net architecture perfectly concatenates deep semantic (meaning) features with shallow spatial (location) features via skip connections.
  • The Weight Problem: Standard 3D convolutions require a catastrophic amount of FLOPs, completely halting edge devices.
  • Our Innovation: Implemented Depthwise Separable 3D Convolutions—splitting the calculation into spatial filtering and 1x1x1 pointwise feature combination.
  • The Result: Semantic volumetric segmentation achieved with only 154,812 parameters.

Advanced Training Dynamics

14
  • Loss Function: Implemented a Combined Loss mechanism summing BCEWithLogitsLoss + Soft Dice Loss to suppress false positives dynamically.
  • Optimizer: AdamW combined with a Cosine Annealing Learning Rate Scheduler for smooth convergence over 50 epochs.
  • The Bias Trick: The final classification layer bias was initialized heavily negative (-5.0) to prevent early training explosive gradients caused by benign-heavy data.
  • Hardware Acceleration: Utilized Automated Mixed Precision (AMP - fp16) on Kaggle Cloud GPUs to halve memory consumption during training.

The PyTorch + Flask Gateway

23
  • Robust Preprocessing (`preprocess_npy`): Automates strict voxel intensity normalization to map all raw Hounsfield units to the absolute [0, 1] range expected by the AI.
